# GitHub 更新指南 (GitHub Update Guide)

本指南將教您如何透過 GitHub 來發布啟動器更新與遊戲內容更新。

## 事前準備

1. 確保您的 GitHub 倉庫名稱為 `ken/minecraft-launcher` (或您在設定中修改的名稱)。
2. 確保該倉庫是 **公開 (Public)** 的 (若為私有，需在程式碼中設定 Token)。

---

## 1. 更新啟動器 (Launcher Self-Update)

當您開發了新功能的啟動器並重新打包成 EXE 後，請依照以下步驟讓玩家自動更新：

1. **前往 GitHub Releases 頁面**
   - 進入您的倉庫頁面。
   - 點擊右側的 "Releases"。
   - 點擊 "Draft a new release"。

2. **建立新版本**
   - **Choose a tag**: 輸入新的版本號，例如 `v1.0.1` (必須比舊版本高，且開頭要有 `v`)。
   - **Release title**: 輸入標題，例如 "v1.0.1 - 修復已知問題"。
   - **Description**: 輸入更新日誌，這會顯示在玩家的更新視窗中。

3. **上傳執行檔**
   - 在 "Attach binaries by dropping them here..." 區域。
   - **重要**：上傳您的 `MCL.exe` (或 JAR 檔)。
   - 檔名必須包含 `.exe` 或 `.jar`。

4. **發布**
   - 點擊 "Publish release"。
   - 玩家下次打開啟動器時，就會看到更新提示並自動下載安裝。

---

## 2. 更新遊戲模組 (Mod Updates)

當您想要新增或刪除模組時，不需要發布 Release，只需更新倉庫檔案：

1. **準備檔案**
   - 在您的電腦上整理好 `mods` 資料夾。

2. **上傳至 GitHub**
   - 將新的模組檔案上傳到倉庫的 `mods/` 目錄中。
   - 刪除舊的模組檔案。
   - **Commit & Push** 這些變更到 `main` 分支。

3. **玩家端行為**
   - 玩家打開啟動器時，程式會自動比對本地與遠端的檔案。
   - 自動下載新模組。
   - 自動刪除舊模組。
   - **注意**：這過程是全自動的，沒有彈窗提示。

---

## 常見問題 (FAQ)

**Q: 為什麼玩家收不到更新？**
A: 請檢查：
   1. 倉庫名稱是否設定正確。
   2. Release 的 Tag 是否真的比玩家當前版本高 (例如 v1.0.1 > v1.0.0)。
   3. 玩家的網路是否能連線到 GitHub API。

**Q: 模組更新太慢怎麼辦？**
A: GitHub API 有速率限制。若模組檔案非常大 (如數百 MB)，建議將模組包成一個 ZIP 放在 Release 中，並修改程式碼改為下載該 ZIP。但目前的檔案同步模式適合頻繁的小更新。
