### 為什麼只要丟 EXE 上去？

因為我們這個啟動器的設計邏輯是這樣的：

1.  **程式碼與模組 (Code & Mods)**：
    *   這些東西我會透過 `git push` 指令直接傳到 GitHub 的「檔案庫 (Code)」區域。
    *   玩家的啟動器會自動去那裡檢查模組有沒有變更。
    *   **所以這部分您不需要手動上傳檔案，我幫您用指令傳好了。**

2.  **啟動器執行檔 (Launcher EXE)**：
    *   這是一個「發布版軟體」，不是原始碼。
    *   GitHub 有一個專門放軟體的地方叫 **"Releases"**。
    *   啟動器的自動更新功能 (`UpdateManager`) 是去檢查 **Releases** 頁面有沒有新的 EXE。
    *   **所以您只需要手動把 `MCL.exe` 拖到 Releases 頁面即可。**

### 總結流程

1.  **我做的事**：
    *   修改設定檔指向您的新倉庫。
    *   用指令把所有程式碼、圖片、模組全部上傳到 GitHub。
    *   重新打包產生最新的 `MCL.exe`。

2.  **您做的事 (最後一步)**：
    *   打開 GitHub 網頁。
    *   建立一個 Release (版本號設為 `v1.0.0`)。
    *   把 `release/MCL/MCL.exe` 上傳上去。
    *   把 `release/MCL` 整個資料夾壓縮成 ZIP，私下傳給朋友 (這是第一次給他們安裝用)。

之後如果有更新，您也只需要重複「建立 Release」並上傳新 EXE 即可，玩家就會自動更新了。
